📄 How Data Is Fetched

fetch_rocm_data.py pulls live data from four GitHub repositories using anonymous sparse git clone for the public repos and an SSH git clone for the private ROCm/InferenceMAX_rocm. No GITHUB_TOKEN is required — ROCm-org now blocks classic PATs, and fine-grained tokens typically need admin approval. Each clone is shallow (--depth=1) and blob-filtered (--filter=blob:none), so the live run finishes in ≈60 seconds and only downloads the handful of files actually used. If any clone fails, the fetcher transparently falls back to the committed JSON snapshots (therock_ci_snapshot.json, inferencemax_snapshot.json).

🔗 Source 5 — therock-runner-health.com (live runner fleet status, AMD-internal)

Populates: the live online/offline/busy/idle counts and queue-health badges shown next to every runner label in the “Server count” section, plus the physical-machine deduplication used to reconcile declared vs. live capacity.

Data source priority (handled by runner_health_parser.load_runner_health_any):
(1) Local TheRock Runner Health.mhtml in this folder — save the dashboard via Save Page As → Webpage Single File while signed in on the AMD network. Not committed to git.
(2a) Anonymous HTTPS GET of the dashboard — 1-second probe; almost always falls through because the URL is gated by GitHub OAuth.
(2b) Playwright with persistent Chromium profile (opt-in dependency: pip install playwright && python -m playwright install chromium) — first run pops a visible Chromium window for one-time GitHub sign-in, every subsequent run reuses the cached session silently and headlessly. Profile lives at ~/.rocm-cicd-report/playwright-profile by default; set RUNNER_HEALTH_NO_PLAYWRIGHT=1 to skip on CI.
(3) runner_health_snapshot.json — committed JSON cache of the last successful parse. Refreshed automatically every time path (1), (2a) or (2b) succeeds, so most users transparently see recent numbers.
Fetch pipeline: Sparse anonymous git clones (public repos) + Sparse SSH clone (InferenceMAX_rocm) → fetch_rocm_data.py → rocm_ci_data.py (intermediate snapshot) → generate_rocm_html.py + generate_rocm_cicd.py → ROCm_CICD_Comprehensive.html + ROCm_CICD_Comprehensive.xlsx
Resilience: any clone failure transparently falls back to the committed therock_ci_snapshot.json / inferencemax_snapshot.json. Runner-health enrichment uses its own three-tier chain (local .mhtml → live HTTPS → committed runner_health_snapshot.json).
